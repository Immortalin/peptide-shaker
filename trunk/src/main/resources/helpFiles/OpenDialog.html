Introduction

The Open dialog allows the opening of identification files. After completing each field click "Open" to start the processing. While importing the progress of the import will be displayed.

From the open dialog, it is also possible to open previously saved projects.

Project Details

The project details will be used to reference the project. It is possible to set the project name, the type of sample and the number of the replicates.

Input Files

PeptideShaker requires information found in three different types of files:

  • Identification File(s): the files containing the identification results. Supported formats are Mascot dat files, OMSSA omx files and X!Tandem xml files.

    It is also possible to load previous PeptideShaker project (cps files) here.

    If the identification files were generated using SearchGUI, PeptideShaker will try to load the other files automatically!

  • Spectrum File(s): the files containing the peak lists used for the search in the mgf format.

    • All spectra should be provided and uniquely entitled! To generate compatible mgf files we recommend the use of OpenMS.

  • FASTA File: the FASTA file used to identify the spectra.

    Identification should be conducted using concatenated Target/Decoy databases. The tag for reverse sequences is "REVERSED". It is recommended to use SearchGUI for the identification of spectra. Peptides common to the Target and the Decoy database will be automatically ignored.

    Loading big files can require a lot of time and memory. We recommend trying PeptideShaker on smaller datasets first!

Processing Parameters

Additional settings can be set for the preprocessing of the identification files (note: not available for cps files):

  • e-value filtering: In order to reduce the memory consumption it is possible to remove identifications with a high e-values. It is adviced to keep these values as high as possible in order to ensure that PSMs reach a PEP of 1. This can be verified in the Validation tab.

  • Peptide Length: lower and upper limits can be set for the peptide length.

  • Precursor Accuracy: The higher precursor mass error allowed can be set here in order to filter out outliers.

  • Search Parameters: The search parameters should be set carefully in order to process the data reliably.

    • If the identification files were generated using SearchGUI, PeptideShaker will try to load the search parameters automatically!
