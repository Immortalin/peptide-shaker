Open Files

Introduction

The Open dialog allows the opening of identification files. After completing each field click on "Open" to start the processing. The progress of the import will then be displayed. From here, it is also possible to open previously saved projects.

Project details

The project details will be used to reference the project. It is possible to set the project name, the type of sample and the number of the replicate.

Input files

PeptideShaker requires information which can be found in three different types of files:

  • Identification File(s): Provide here files containing the identification results. It is possible to open Mascot .dat files, OMSSA .omx files and X!Tandem .t.xml files.
    From here it is also possible to load a former PeptideShaker project (.cps file).

    • If the identification files were generated using     SearchGUI, PeptideShaker will try to load the other files automatically!

  • Spectrum File(s): Provide here files containing the peak lists used for the search in the mgf format.

    • All spectra should be provided and uniquely entitled! In order to generate compatible mgf files we recommend the use of OpenMS.

  • FASTA File: Provide here the FASTA file used to identify the spectra.

    • Identification should be conducted using concatenated Target/Decoy databases. The tag for reverse sequences is "REVERSED". It is recommended to use SearchGUI for the identification of spectra. Peptides common to the Target and the Decoy database will be automatically ignored.

    Loading the files can require a lot of time and memory. We recommend to try PeptideShaker first with small datasets!

Processing Parameters

Additional settings can be set for the preprocessing of the identification files (not available for cps files):

  • e-value filtering: In order to reduce the memory consumption it is possible to remove identifications with a high e-value. It is adviced to keep these values as high as possible in order to ensure that PSMs reach a PEP of 1. This can be verified in the validation panel.

  • Peptide Length: lower and upper limits can be set for the peptide length.

  • Precursor accuracy: The higher precursor mass error allowed can be set here in order to filter out outliers.

  • Search Parameters: The search parameters should be set carefully in order to process the data reliably.

    • If the identification files were generated using SearchGUI, PeptideShaker will try to load the search parameters automatically!
